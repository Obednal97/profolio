## Definitions

To ensure clarity in these guidelines, the following terms have specific meanings:

- **"Community"** means all users, contributors, and participants in the Profolio ecosystem
- **"Content"** means any data, text, files, code, information, or materials shared within the platform or community spaces
- **"Contribution"** means code, documentation, issues, discussions, or any other input to the project
- **"Platform"** means the Profolio portfolio management system and all associated community spaces
- **"Prohibited Content"** means any content that violates these guidelines or applicable laws
- **"User"** means any individual or entity accessing or contributing to the platform

## 1. Purpose and Scope

1.1. **Application**

These Guidelines apply to all interactions within:

- The Profolio platform (cloud and self-hosted versions)
- GitHub repositories and discussions
- Community forums and support channels
- Any official Profolio communication channels
- Third-party platforms when representing Profolio

  1.2. **Our Mission**

Profolio is committed to providing:

- **Privacy-First Solutions:** Your financial data under your control
- **Open Source Excellence:** Transparent, auditable, community-driven development
- **Professional Standards:** Enterprise-grade functionality for personal use
- **Inclusive Community:** Welcoming environment for all skill levels

## 2. General Content Standards

2.1. **Acceptable Content and Behaviour**

All users must maintain professional standards by:

- Contributing constructive, relevant content to discussions
- Respecting intellectual property rights
- Providing accurate information when sharing knowledge
- Following security best practices when sharing code
- Maintaining confidentiality of sensitive information

  2.2. **Quality Standards**

When contributing to the platform:

- Write clear, well-documented code
- Test thoroughly before submitting changes
- Follow established coding conventions
- Provide helpful commit messages and PR descriptions
- Respond constructively to feedback

  2.3. **Communication Standards**

All communication must be:

- Professional and respectful
- Free from offensive language
- Focused on technical merit
- Inclusive of diverse perspectives
- Constructive in criticism

## 3. Prohibited Content and Behaviour

### 3.1 Harassment and Discrimination

**Strictly Prohibited:**

- Personal attacks or threats against any individual
- Discriminatory language based on protected characteristics
- Doxxing or sharing private information without consent
- Stalking or persistent unwanted contact
- Sexual harassment or inappropriate advances

### 3.2 Malicious Activities

**Zero Tolerance for:**

- Uploading malware, viruses, or malicious code
- Attempting unauthorised access to systems or accounts
- Data scraping or harvesting without permission
- Denial of service attacks or system disruption
- Exploiting vulnerabilities without responsible disclosure

### 3.3 Fraudulent and Deceptive Practices

**Not Permitted:**

- Impersonation of individuals or organisations
- Spreading misinformation or false technical advice
- Market manipulation or financial fraud
- Phishing or social engineering attempts
- Creating multiple accounts to evade restrictions

### 3.4 Spam and Commercial Exploitation

**Prohibited Activities:**

- Unsolicited commercial messages or advertisements
- Excessive self-promotion without community value
- Link farming or SEO manipulation
- Automated posting or bot activity without approval
- Using the platform for unauthorised commercial purposes

## 4. Privacy and Data Protection

4.1. **Personal Information**

Users must not:

- Share others' personal or financial information without consent
- Request sensitive personal data in public forums
- Post screenshots containing private information
- Bypass privacy settings or access controls

  4.2. **Platform Security**

All users must:

- Report security vulnerabilities through proper channels
- Avoid publicly disclosing unpatched vulnerabilities
- Use secure communication for sensitive topics
- Respect rate limits and API quotas
- Follow responsible disclosure practices

## 5. Intellectual Property

5.1. **Respecting Rights**

Users must:

- Only share content they have rights to distribute
- Provide proper attribution for others' work
- Respect software licences and terms
- Not reverse engineer proprietary components
- Comply with open source licence requirements

  5.2. **Contributing Content**

By contributing, you:

- Grant necessary licences for your contributions
- Confirm you have rights to the content
- Agree to the project's licence terms (MIT)
- Allow the community to build upon your work

## 6. Enforcement and Moderation

### 6.1 Enforcement Actions

Violations may result in:

**Warning**

- First-time minor violations
- Unintentional guideline breaches
- Good faith mistakes

**Temporary Restriction**

- Repeated minor violations
- Single serious violation
- Pending investigation of reports

**Permanent Ban**

- Severe violations (malware, fraud, harassment)
- Repeated violations after warnings
- Endangering community safety

### 6.2 Reporting Violations

To report guideline violations:

1. Use the report function on the platform
2. Email security@profolio.app for security issues
3. Contact moderators for sensitive matters
4. Include evidence and context in reports

### 6.3 Appeals Process

If you believe action was taken in error:

- Contact appeals@profolio.app within 30 days
- Provide explanation and any mitigating factors
- Allow up to 7 days for review
- Decisions are reviewed by multiple moderators

## 8. Recognition and Rewards

8.1. **Contributor Recognition**

We celebrate community contributions through:

- Contributors list in releases
- Featured community spotlights
- Maintainer invitations for regular contributors
- Public acknowledgment in announcements

  8.2. **Types of Contributions**

All contributions are valued:

- **Code:** Features, bug fixes, optimizations
- **Documentation:** Guides, tutorials, translations
- **Testing:** Bug reports, test coverage, QA
- **Support:** Helping others, answering questions
- **Design:** UI/UX improvements, graphics
- **Advocacy:** Spreading awareness, writing articles

## 9. Open Source Commitments

9.1. **Transparency**

We commit to:

- Open development process
- Public roadmap and planning
- Clear communication about changes
- Documented decision-making

  9.2. **Licensing**

- All code under MIT License
- Documentation under Creative Commons
- Contributors retain copyright
- No proprietary lock-in

  9.3. **Sustainability**

Supporting the project:

- Star and watch the repository
- Contribute code or documentation
- Share with your network
- Sponsor development (optional)

## 10. Platform-Specific Guidelines

### 10.1 GitHub Repository

When using our GitHub spaces:

- Use issue templates for reports
- Search before creating new issues
- Provide minimal reproducible examples
- Follow PR contribution guidelines
- Use appropriate labels

### 10.2 Self-Hosted Deployments

For self-hosted users:

- You control your data completely
- Community guidelines still apply to contributions
- Share improvements back to the community
- Report bugs that affect all users

## Quick Reference Guide

### ✅ Do's

- **Contribute** constructively to discussions
- **Help** other community members
- **Report** bugs and security issues responsibly
- **Follow** coding standards and conventions
- **Respect** different viewpoints and experiences
- **Document** your contributions clearly
- **Test** your changes thoroughly

### ❌ Don'ts

- **Harass** or discriminate against others
- **Share** private information without consent
- **Upload** malicious code or malware
- **Spam** or advertise without permission
- **Violate** intellectual property rights
- **Exploit** platform vulnerabilities
- **Impersonate** other users or organisations

### 🚀 Getting Help

- **Documentation:** Start with our comprehensive docs
- **Discussions:** Ask questions in GitHub Discussions
- **Issues:** Report bugs with detailed information
- **Security:** Email security@profolio.app privately

### 📊 Enforcement Levels

1. **Warning** - Minor first offences
2. **Restriction** - Repeated or serious violations
3. **Ban** - Severe or persistent violations
